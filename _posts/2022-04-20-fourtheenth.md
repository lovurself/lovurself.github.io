웹페이지 제작할 때 웹표준에 맞추어 크로스 브라우징이 되도록 해야 한다. 그래야 어떠한 운영체제든 브라우저든 내가 의도한 대로 보여줄 수 있기 때문이다.

웹표준에 맞게 만들려면 제일 기초인 HTML 문서를 작성할 때 시맨틱 태그를 사용하는 것을 권장한다.



### 시맨틱 태그

= Semantic Tag 의미의 태그

시맨틱 태그는 의미가 있는 태그라는 말로 바꾸어 말할 수 있다. 사람들도 서로 어떤 말을 하는지 정확하게 정해놓으면 소통하기 쉬운 것처럼 브라우저도 정확하게 어떤 의미인지 아는 태그를 사용해서 HTML 문서를 작성하면 크로스 브라우징에 좋다.

여기서 의미없는 태그에는 div, span 태그처럼 class나 id를 내가 작성해서 넣어야 의미가 생기는 태그가 있다. 그럼 시맨틱 태그의 종류와 그 태그를 어떻게 사용할 수 있는지 정리해보려고 한다.



**시맨틱 태그의 종류와 쓰임**

1. header   

   => 제목, 로고, 검색 폼 등을 포함할 수 있고, 보통 웹페이지의 최상단에 위치시킨다. 하지만 전역특성이여서 일부 태그 외에는 공통으로 사용할 수 있다.

2. nav   

   => 메뉴, 목차, 색인 등으로 사용할 수 있고, 전역특성으로 하나의 문서 안에 여러개의 nav 태그를 사용할 수 있다. 현재 페이지의 요소들의 위치를 찾을 수도 있고, 다른 페이지로 이동할 수도 있다.

3. main   

   => 문서에서 주요 콘텐츠를 나타내는 태그이다. 문서의 핵심 주제나 앱의 핵심 기능에 직접적으로 연결되었거나 확장하는 콘텐츠로 구성할 수 있다. 전역특성이지만 hidden 속성없이는 문서 내에 여러개의 main 요소가 존재하면 안된다.

4. section   

   => (무조건적인 것은 아니지만) main 태그 내에 section 태그를 사용해서 콘텐츠를 구분할 수 있다. 그리고 사진처럼 header 태그를 포함하여 제목을 보여줘도 되지만 필수적인 것은 아니다. 대신 h1~h6 태그를 사용하여 제목을 만들 수 있다. MDN에서는 section 태그를 일반 컨테이너로 사용하지 않는 것을 권장한다. 문서 요약에 해당 구획이 논리적으로 나타낼 때 사용하기를 권장한다.

5. article   

   => 게시판, 블로그 글, 매거진, 뉴스 기사 등을 보여줄 때 사용하는 태그이다. article 태그 안에 또 article 태그를 중첩 사용 가능하다.

6. footer   

   => footer 태그에는 작성자, 저작권 정보, 회사정보, 연락처 등을 보여준다. 



HTML 문서의 뼈대가 되는 태그들을 정리해보았고 더 많은 태그들이 있으니 **[MDN web docs](https://developer.mozilla.org/ko/)** 에서 확인하고 공부하면 된다.



### SEO

= Search Engine Optimization 검색 엔진 최적화

검색 엔진으로부터 웹사이트나 웹페이지에 대한 웹사이트 트래픽의 품질과 양을 개선하는 과정이다. 

평소 내가 알고 싶은 내용을 검색을 하면 상단에 있는 링크에 들어가게 된다. 그럼 검색 결과 상단에 나올 수 있게 하는 방법은 무엇일까? 검색엔진은 자료를 수집하고 순위를 매겨서 검색 결과 상단에 나오게 하는데, 이때 웹페이지를 순위를 매기는 방식에 맞게 구성하면 된다. 

구글 등장 이후 검색 엔진들이 콘텐츠의 신뢰도를 파악하는 기초적인 지표로 다른 웹사이트에 얼마나 인용되었나를 사용하기 때문에 타 사이트에 인용되는 횟수를 늘리는 방향으로 최적화한다.

### 시맨틱 태그와 SEO의 관계

내가 만든 웹페이지를 사람들이 많이 들어오게 만드려면 사람들이 검색을 했을 때 검색 결과 상단에 있어야 많이 들어올 것이다. 그럼 내 웹페이지를 검색 결과 상단에 올려야 하는데, 위에서 말했듯이 검색 엔진이 순위를 매기는 방식에 맞게 웹페이지를 구성하면 된다. 그게 바로 시맨틱 태그를 사용하는 것이다.

검색엔진은 웹페이지 내의 검색 키워드 순위를 태그로 판단을 한다. 그러므로 올바르고 의미있는 태그를 사용하여 검색엔진에 최적화시켜야 한다.
